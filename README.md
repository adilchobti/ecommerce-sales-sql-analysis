# E-commerce Sales Analysis (SQL)

## Description
Projet d’analyse SQL des ventes e-commerce à partir d’un dataset transactionnel réel.  
L’objectif est de structurer les données, écrire des requêtes SQL analytiques et produire des visualisations afin d’extraire des indicateurs business clés.

---

## Objectifs du projet
- Nettoyer et préparer des données transactionnelles brutes  
- Concevoir une structure de données exploitable pour l’analyse  
- Réaliser des analyses SQL orientées business  
- Visualiser les résultats pour faciliter l’interprétation  

---

## Dataset
- Source : Online Retail II  
- Période : ventes e-commerce  
- Données principales :
  - Commandes (factures)
  - Produits
  - Quantités et prix
  - Clients
  - Pays

---

## Technologies utilisées
- SQL (SQLite)
- Python
- pandas
- matplotlib
- Google Colab

---

## Analyses réalisées
Les requêtes SQL permettent notamment de :

- Analyser l’évolution du chiffre d’affaires dans le temps  
- Identifier les produits générant le plus de chiffre d’affaires  
- Segmenter les clients selon leur fréquence d’achat :
  - Clients occasionnels
  - Clients réguliers
  - Clients premium

Les résultats sont ensuite visualisés sous forme de graphiques pour une lecture plus intuitive.

---

## Visualisations
- Évolution du chiffre d’affaires mensuel  
- Top 10 des produits par chiffre d’affaires  
- Répartition des clients par segment  

---

## Comment exécuter le projet
1. Ouvrir le notebook dans Google Colab  
2. Importer le fichier `online_retail_II.xlsx`  
3. Exécuter toutes les cellules  
4. Les graphiques sont générés automatiquement et sauvegardés en images  

---

## Ce que montre ce projet
- Capacité à structurer et analyser des données avec SQL  
- Compréhension des enjeux business (ventes, clients, performance produit)  
- Capacité à transformer des résultats SQL en visualisations exploitables
